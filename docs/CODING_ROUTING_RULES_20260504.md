# 智能开票系统｜识别与赋码路由规则（2026-05-04）

## 1. 文本 / 附件结构化识别是否调用 LLM

### 纯文本输入

默认优先使用本地规则解析购买方、税号、明细、金额和税率。

如果本地规则已经形成完整草稿：

- 购买方名称存在
- 购买方税号存在
- 至少 1 行明细
- 每行有项目名称
- 每行有含税金额

则默认走快速路径：

```text
rules_only
```

此时不调用结构化识别 LLM。这样做是为了避免简单文本每次都等待大模型。

如果纯文本存在以下情况，则调用文本 LLM：

- 规则无法识别购买方名称或税号
- 规则无法识别明细行
- 规则无法识别金额
- 文本较长或自然语言较复杂
- 环境变量 `TAX_INVOICE_LLM_BLOCKING_REVIEW=1/true/on/yes` 强制开启复核

### 单张模式 + 附件

未勾选“批量开具发票”时，只要上传附件，就强制进入 LLM 复核链路：

```text
force_llm_review=True
```

图片优先走视觉 LLM；PDF / Excel / 文档先提取文字，再进入文本 LLM 结构化复核。

### 批量模式 + 图片

勾选“批量开具发票”且图片数量 1-5 张时：

```text
一张图片 = 一个子草稿
逐张调用视觉 LLM
strategy = rules_plus_batch_vision
```

超过 5 张时回到 OCR / 文档解析 / 本地规则 / 待补全混合模式。

## 2. 品类明细 → 赋码库映射顺序

识别出明细行以后，赋码不是简单按主体历史高频项直接填入，而是按以下顺序处理。

### Step 1：保护材料中已经明确给出的字段

如果材料/LLM/Excel 已经给出：

- 项目名称
- 税收分类
- 税收编码
- 税率

系统优先保留这些字段。后续规则只能补空字段，不能随意覆盖明确字段。

### Step 2：客户同步赋码规则 / 本地即时学习规则 / 内置基础规则

优先匹配当前明细行的项目名称。

安全匹配包括：

- 项目名称与别名完全一致
- 项目名称包含别名
- 别名扩展了项目名称，且项目名称不是泛化词

不安全匹配：

- 仅因为整段材料里出现过某个历史别名，就把该别名套到明确明细行上

例如：

```text
当前明细：A4复印纸 / 蓝色文件夹 / 桌面文件架
材料备注或历史测试词：代理记账和税务申报
```

不能把 3 行文具赋为“代理记账”。

上下文匹配只允许用于空项目或泛化项目，例如：

```text
服务费
商品
材料费
货物
```

### Step 3：本地安全规则

对已知高风险/高频项目有本地安全映射，例如：

```text
A4复印纸 / 打印纸 / 复印纸 → 纸制品 / 1060105020000000000
文件夹 → 文具 / 1060401020000000000
文件架 → 文具 / 1060401030000000000
注射器 / 输液针 / 穿刺针 → 医疗仪器器械 / 1090245030000000000
医用丁腈/乳胶手套 → 橡胶制品 / 1070508010000000000
```

这些规则的定位是：现场演示和早期客户测试中，先保证明显品类不会被历史服务类项目污染。

### Step 4：官方分类库候选

如果本地规则未命中，则调用官方税收分类库搜索。

- 候选分数 >= 84：可填入税收分类候选
- 候选分数 >= 95：可填入税收编码

仍标记为候选/需复核。

### Step 5：税收编码 LLM 候选

只有在以下条件同时满足时，才调用税收编码 LLM：

- 当前行仍缺税收分类或税收编码
- 官方分类库能给出候选集合
- LLM 配置启用且可用

LLM 只在候选集合中选择，不直接创造最终编码。

结果标记为：

```text
智能推荐，需人工复核
```

### Step 6：主体历史档案 / 常用项目

主体历史档案只能用于：

- 购买方历史补全
- 弱项目名补全，例如“服务费”“代理服务”
- 批量页“可编辑推荐框”的预填

不能直接覆盖明确明细，例如：

```text
A4复印纸
蓝色文件夹
桌面文件架
餐费
```

批量页的历史常用项目只能作为可编辑推荐：助理确认或修改后，再一键导入到 Sheet2。

## 3. 当前修复重点

本次针对录屏中出现的“文具明细被赋为代理记账”风险，已明确：

```text
整段材料 / 备注 / 历史测试词中出现的别名，不得覆盖当前明确项目名称。
```

也就是说，即使上下文里出现“代理记账和税务申报”，只要当前明细行是：

```text
A4复印纸
蓝色文件夹
桌面文件架
```

系统必须按明细本身走办公用品 / 文具规则，不能套用代理记账。

## 4. 录屏解释口径

可以这样对客户解释：

```text
系统先识别购买方和明细，再按明细内容匹配赋码规则。
历史开票习惯只作为辅助推荐，不会直接覆盖当前明细。
遇到新明细时，系统会先给出规则/分类候选；没有把握的地方会提示复核。
```

不要承诺：

```text
所有新明细都能自动准确赋码
完全不需要助理复核
历史项目可以自动套到所有新票
```
