# 2026-05-08 沈阳现场部署与助理培训 SOP

## 一、现场目标

本次现场目标不是追求“完全无人值守开票”，而是验证助理能稳定完成：

1. 首次安装与私密配置安装。
2. 启动智能开票助手。
3. 登录税局专用浏览器。
4. 上传材料或粘贴文本。
5. 生成草稿 / 批量工作表。
6. 人工复核购买方、项目、金额、税率、税收编码。
7. 下载或上传税局模板。
8. 到税局“预览发票”页面。
9. 不点击最终开具。
10. 在执行页人工确认并形成执行记录。

## 二、不要讲的技术词

现场给助理培训时尽量不讲：

- OCR
- LLM
- 大模型
- 云端档案
- 规则引擎
- CDP
- telemetry

统一换成助理能理解的说法：

- 识别材料
- 生成草稿
- 人工复核
- 上传税局
- 到预览页
- 执行记录
- 主体与档案

## 三、首次部署步骤

### 1. 解压公开安装包

建议路径：

```text
C:\智能开票助手\
```

如果现场电脑不能稳定处理中文路径，可先用：

```text
C:\invoice-assistant\
```

### 2. 安装私密配置

双击：

```text
01_INSTALL_PRIVATE_CONFIG.bat
```

预期：提示安装成功。

### 3. 启动工作台

双击：

```text
02_START_INVOICE_ASSISTANT.bat
```

预期出现两个窗口：

1. 智能开票助手工作台：

```text
http://127.0.0.1:5012
```

2. 税局专用 Edge 浏览器：

```text
about:blank
```

注意：空白 Edge 是正常的，不能关闭。

## 四、现场主流程

### 1. 打开税局

在工作台点击对应按钮：

- 打开辽宁税局
- 打开吉林税局
- 打开黑龙江税局
- 打开北京税局

税局登录页应该在税局专用 Edge 窗口打开。

### 2. 登录税局

由客户 / 助理人工登录。

### 3. 识别当前税局主体

点击：

```text
识别当前税局主体 / 加载档案
```

确认当前税局主体显示正确。

### 4. 输入材料

支持：

- 微信自然语言文本
- Excel
- PDF
- 图片 / 截图
- 餐饮平台客户发票截图

如果是多张截图 / 多个购买方 / 需要批量模板，必须勾选：

```text
批量开具发票
```

### 5. 生成草稿

点击：

```text
生成开票草稿
```

### 6. 人工复核

重点复核：

- 销售方主体
- 购买方名称
- 购买方税号
- 项目名称
- 含税金额
- 税率
- 税收分类
- 税收编码

### 7. 发起税局执行

点击：

```text
启动开票
```

提交前系统会检查：

```text
当前税局登录主体 = 草稿 / 批量草稿销售方
```

如果不一致，系统会禁止提交。

### 8. 到预览页

系统只辅助到税局预览页。

明确不要点击：

```text
最终开具
```

### 9. 人工确认执行记录

税局执行页完成后，由助理点击：

```text
已人工确认，记录为成功
```

该记录进入“执行记录”，用于现场复盘和后续产品优化。

## 五、执行记录怎么用

“执行记录”不是客户历史发票档案。

它用于回答：

- 这次从什么材料开始？
- 识别策略是什么？
- 助理改了哪些字段？
- 模板有没有阻断错误？
- 税局是否执行成功？
- 是否到预览？
- 是否人工确认？
- 如果失败，失败在哪一步？

客户历史发票、购买方档案、常用项目仍以“主体与档案”为准。

## 六、遇到问题时的反馈模板

如果助理遇到问题，让助理尽量按以下格式反馈：

```text
1. 当前在哪个页面？
   开票工作台 / 草稿页 / 批量复核 / 税局执行 / 主体与档案 / 执行记录

2. 执行记录 case_id 是什么？

3. 材料类型是什么？
   文本 / Excel / PDF / 图片 / 多张截图

4. 是否勾选了批量开具发票？

5. 当前税局登录主体是谁？

6. 草稿销售方主体是谁？

7. 页面提示了什么错误？

8. 税局页面有没有截图？

9. 是否到达预览发票？

10. 是否点击了人工确认？
```

## 七、常见问题判断

### 1. 出现两个窗口是不是正常？

正常。

- 工作台窗口负责上传材料、复核、执行。
- 空白 Edge 是税局专用浏览器。

### 2. Edge 一开始是空白是不是错误？

不是错误。

点击工作台里的“打开 XX 税局”后，税局页面会在这个 Edge 中打开。

### 3. 吉林商贸没有历史发票，主体与档案显示暂无数据是不是失败？

不是失败。

这是安全空状态：

- 不建空档案
- 不同步空数据
- 不影响继续开票主流程

### 4. 主体不一致为什么不让提交？

这是安全保护。

如果当前税局登录主体和草稿销售方不一致，系统必须阻断，避免登录 A 主体却给 B 主体提交开票。

### 5. 执行记录是否会进入客户档案？

不会。

执行记录只用于产品排障和现场复盘，不作为客户历史发票档案来源。
